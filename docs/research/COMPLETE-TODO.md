# Complete To-Do — Research + System + Next Session

**Purpose:** Single sequenced checklist to complete the research, continue system development, and run the next MSWDO session (hands-on for staff → pilot with staff + beneficiaries).

**References:** [session-1-record.md](session-1-record.md), [MSWDO-VISIT-STRUCTURE.md](MSWDO-VISIT-STRUCTURE.md), [README.md](README.md)

---

## Phase 1 — Before the Next Session (Preparation)

| # | Task | Notes |
|---|------|--------|
| 1 | **Confirm pilot date** | When Ma'am La Arni messages; lock the date and document it. |
| 2 | **Coordinate with MSWDO IT** | Technical discussion (config, deployment, program settings) as agreed with Ma'am La Arni; align server/Wi‑Fi and devices for pilot. |
| 3 | **Thematic analysis of Session 1 interview** | Code Session 1 Q&A into themes (manual process, bottlenecks, program steps, privacy, environment) for Ch3 and Objective 1. |
| 4 | **System work before next session** | Fix known bugs; implement small improvements from Session 1 if any; ensure AICS-ready program + tracks + stations + tokens; test triage → station → informant flow end-to-end. |
| 5 | **Prepare for next session** | Decide format: one long session (hands-on then pilot same day) or two visits (2a = hands-on, 2b = pilot day). Prepare: consent (if new respondents), feedback form, SUS copies, training/quick-reference materials, token batch for pilot. |
| 6 | **Confirm attendees and agenda** | With MSWDO: who joins hands-on (staff + admin? IT?); who is on-site for pilot (staff + beneficiaries); duration and venue. |

---

## Phase 2 — Next Session: Hands-On Then Pilot (Staff + Beneficiaries)

**Options:**  
- **Same day:** Hands-on for staff → pilot with staff + beneficiaries in one session.  
- **Two visits:** Session 2a = hands-on for staff; Session 2b = pilot day with staff + beneficiaries.

### Suggested order (if same day)

| # | Block | Purpose |
|---|--------|--------|
| 1 | Opening | Thanks, agenda: hands-on → pilot → SUS. |
| 2 | **Hands-on (staff)** | Staff try Triage + Station (and Admin dashboard if relevant). Support and note issues/feedback. |
| 3 | Structured feedback (staff) | Short discussion + feedback form (workflow, labels, mobile, informant). |
| 4 | **Pilot (staff + beneficiaries)** | Staff run queue; beneficiaries get tokens (or use pre-printed), go through flow; beneficiaries use informant/scan. |
| 5 | **SUS** | Administer SUS to staff (and admin) who used the system. |
| 6 | Closing | Pilot date confirmation (if not already set), next steps, hand off materials. |

---

## Phase 3 — After the Next Session (Research + System)

| # | Task | Notes |
|---|------|--------|
| 1 | **Transcribe and code feedback** | From hands-on and pilot; merge with Session 1 themes for Ch3. |
| 2 | **Compute SUS** | Score SUS (Objective 3); item-level and mean; interpret per manuscript Table 3. |
| 3 | **Document Session 2** | Create session-2 record: who attended, hands-on summary, pilot summary, SUS administration, any changes agreed. |
| 4 | **Ch3 (Results)** | Present Session 1 (interview themes, requirements); Session 2 (hands-on/pilot feedback, SUS results, pilot outcomes). |
| 5 | **Ch4 (Discussion / Conclusion)** | How requirements and user involvement informed design (RAD); limitations; recommendations. |
| 6 | **Consent and ethics** | Ensure all respondents (Session 1 + Session 2) are covered for manuscript/ethics. |
| 7 | **System refinements** | Implement high-priority fixes from hands-on and pilot feedback; document “changes made based on Session 2.” |

---

## One-Page Sequence Summary

1. **Before next session:** Confirm pilot date; MSWDO IT coordination; Session 1 thematic analysis; system stable and AICS-ready; prep materials (SUS, feedback form, tokens, training); confirm attendees and agenda (hands-on then pilot, same day or two visits).
2. **Next session:** Hands-on for staff (Triage + Station + optional Admin) → feedback → Pilot with staff + beneficiaries → SUS (staff/admin) → closing and pilot confirmation.
3. **After:** Code feedback; compute SUS; document Session 2; write Ch3 and Ch4; consent/ethics; system refinements from feedback.
