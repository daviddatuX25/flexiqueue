# QR Token Print — Technical Spec

**Purpose:** Template dimensions, cut lines, and print settings for token card output. Per `docs/plans/QR-TOKEN-PRINT-SYSTEM.md`.

---

## 1. Cut Lines

- **Definition:** Dashed borders (`border: 1px dashed`) around each token card.
- **Use:** Cut along these lines to separate individual cards from the printed sheet.
- **Visibility:** Printed by default; ensures accurate cutting on perforated or hand-cut cards.

---

## 2. QR Code Size

- **Minimum:** ~20mm (2cm) for reliable scanning per plan §4.4.
- **Current:** 32mm × 32mm (generated by `TokenPrintService`).
- **Rationale:** Sufficient margin for scan reliability across typical phone cameras.

---

---

## 5. Background Image (Per-Token Card)

- **Recommended dimensions:** 60mm × 50mm (W×H), or **6:5 aspect ratio** (1.2:1)
- **Rationale:** Matches default card cell size; grid layout yields ~60mm width and ~50mm height per card
- **Min resolution:** 300×250 px for print quality (~120 DPI)
- **Behavior:** Image is applied as `background-size: cover` to fill each card; content (logo, ID, QR, hint, footer) overlays on top

---

## 6. Print Settings (Admin UI)

Template options are stored in `print_settings` and editable via the Tokens page:

- **Cards per page:** 4–8
- **Paper:** A4, Letter
- **Orientation:** Portrait, Landscape
- **Show hint:** Toggle "Scan for status" on cards
- **Show cut lines:** Toggle dashed borders
- **Logo URL:** Optional logo above each card
- **Footer text:** Optional text shown on each token card (below hint), centered
- **Background image URL:** Optional; use 6:5 ratio for best fit (see §5)

**Flow:** Select tokens → Print selected → Modal with options (pre-filled from saved settings) → Print (opens with params) or Save as default.

## 7. Print Instructions (UI)

The Print page (`Admin/Tokens/Print.svelte`) provides a collapsible "Print instructions" section that documents:

- Cut lines purpose
- Paper size (A4/Letter)
- QR scan reliability and sample testing
